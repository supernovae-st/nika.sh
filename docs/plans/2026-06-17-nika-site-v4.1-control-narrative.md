## 1. The pivot (why this rewrite exists)

The v4 site sold the **vision/standard** — "Intent as Code · an open language for
AI workflows." That's the *after-the-fact* pitch; a cold visitor thinks *"I have
LangGraph / n8n / Cursor / MCP — why does this exist?"*

We now sell the **burning problem + control**:

> **AI agents are getting real power — read your repo, call an API, modify code,
> run a script. Today they act from hidden prompts, chat history, closed
> platforms. A black box. Nika makes the agent write its plan as a readable file
> FIRST — the steps, the tools, the permissions, the outputs. A human reviews it.
> The runtime enforces it. Then it runs — traced and replayable.**

The enemy is **the agentic black box**. Nika is **the seatbelt for agents** —
"the control layer between *the model wants to act* and *the system lets it act*."
NOT another agent framework.

## 5. The demo workflow (the control example) — REQUIREMENTS
Short (~12–20 readable lines), **spec-valid (projected from nika-spec)**, and it must SHOW control:
- a `permits:` block (the seatbelt — fs read XOR write, net hosts, tools),
- an `agent` and/or `infer` doing something **real + review-worthy** (e.g. PR review: read a diff → judge risk → comment **only if** high-risk; or "open a PR only if tests pass"; or "page on-call **only if** severity high"),
- a conditional **gate** (`when:`),
- typed outputs.
**LOCKED → fil-rouge = `t3-resume-screener`** (the ONLY projected showcase workflow with a real `permits:` block; `SHOWCASE_YAML['t3-resume-screener']`). Story: a LOCAL model (`ollama/llama3.1`) screens CVs; the file declares `permits: { fs: { read: ["./hiring/inbox/**"], write: ["./hiring/out/**", …] }, tools: [4 data-safe tools] }` and **no `net:` at all** → the PII physically cannot leave the machine, even if a CV hijacks the model via prompt injection. Visceral + concrete (hiring · PII · local · injection). We do NOT reproject (the generated file carries the operator's pending canon-sync edits — do not touch it); use the file as already projected.
- **Hero**: show a FOCUSED real excerpt (the `nika: v1`/`workflow`/`description` + the `permits:` block + the key `infer` scoring task + the `when:` gate) — real lines sliced from the real file, with a "view the full file →" link to `/use-cases` or `/play`. Not hand-typed; a true slice.
- **Living File**: use the full `SHOWCASE_DAG['t3-resume-screener']` (richer DAG = a deeper corridor). The **ENFORCE** beat shows a denied action via the run-model's `failAt` → **`NIKA-SEC-004`** ("effect outside the declared `permits` boundary — denied"). The `permits:` block is the star of the REVIEW beat.
- **The "denied" path** is real: `NIKA-SEC-004` (permits boundary), `NIKA-SEC-001` (exec blocklist), `NIKA-SEC-002` (agent tool outside the whitelist), from `public/errors/catalog.json`.
